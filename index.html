Teste de Validação HTML com Regex

Teste os atributos pattern e title para validação de formulários

Regex: [0-9]{4}

Valida exatamente 4 dígitos numéricos.

Regex: [A-Za-zÀ-ÿ]{3,}

Valida pelo menos 3 caracteres alfabéticos (incluindo acentos).

Regex: [a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$

Valida um formato de e-mail padrão.

Regex: (\([0-9]{2}\)\s?)?[0-9]{4,5}-?[0-9]{4}

Valida números de telefone brasileiros com ou sem DDD.

Regex: [0-9]{3}\.?[0-9]{3}\.?[0-9]{3}-?[0-9]{2}

Valida o formato de CPF brasileiro (com ou sem pontuação).

Regex: ^(?=.*[A-Z])(?=.*\d).{8,}$

Valida senha com pelo menos 8 caracteres, uma letra maiúscula e um número.

Regex: (0[1-9]|[12][0-9]|3[01])/(0[1-9]|1[0-2])/[0-9]{4}

Valida datas no formato brasileiro (DD/MM/AAAA).

Regex: [0-9]{5}-?[0-9]{3}

Valida CEP brasileiro com ou sem hífen.

Suporte do Navegador

Os atributos pattern e title são suportados nos seguintes navegadores:

Chrome 10+ Firefox 4+ Safari 10.1+ Edge 12+ IE 10+

Nota: A pseudo-classe :placeholder-shown é suportada no Chrome 47+, Firefox 51+, Safari 9+, Edge 79+.
